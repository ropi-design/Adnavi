# ステップ 2: キューワーカーの起動

## 📝 キューワーカーとは？

レポート生成などの重い処理をバックグラウンドで実行するための仕組みです。

## 🚀 起動方法

### 新しいターミナルを開く

**重要**: 現在のターミナルとは別の新しいターミナルウィンドウを開いてください。

### キューワーカーを起動

新しいターミナルで以下を実行：

```bash
cd /Users/satohiro/camp/100_laravel/Adnavi
./vendor/bin/sail artisan queue:work
```

### 正常起動の確認

以下のように表示されれば OK：

```
INFO  Processing jobs from the [default] queue.
```

このターミナルは**閉じないでください**。バックグラウンドで動き続けます。

## ⚠️ 注意事項

1. **ターミナルを閉じない** - キューワーカーが停止します
2. **常に起動しておく** - レポート生成時に必要
3. **停止したい場合** - `Ctrl + C`で停止

## 🔄 再起動が必要な場合

.env を変更した後は、キューワーカーを再起動：

```bash
# 現在のワーカーを停止（Ctrl + C）
# 再度起動
./vendor/bin/sail artisan queue:work
```

## 📊 ログの確認

別のターミナルでログを監視（オプション）：

```bash
./vendor/bin/sail artisan pail
```

または

```bash
tail -f storage/logs/laravel.log
```

## ✅ 完了したら

キューワーカーが起動したら「起動した」と教えてください。
次はテストでレポート生成を試します！
